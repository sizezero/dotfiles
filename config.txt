
This is the documentation for how to version control dotfiles across
multiple systems via git and the config alias.

https://developer.atlassian.com/blog/2016/02/best-way-to-store-dotfiles-git-bare-repo/

To show all files in the dotfiles repo:
config ls-files

To use the dotfiles on a new installation peform the following:

git clone --bare https://github.com/sizezero/dotfiles.git ~/.cfg
alias config='/usr/bin/git --git-dir=$HOME/.cfg/ --work-tree=$HOME'
config checkout
<if failed, delete or backup any files and retry>
config config --local status.showUntrackedFiles no
